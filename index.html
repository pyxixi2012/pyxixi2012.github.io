Setting up ElasticSearch with MongoDB

1. Install and Config MongoDB replcate set

prepare install

$ mkdir /data/app && cd /data/app

$ wget https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-rhel70-3.2.5.tgz

$ tar zvxf mongodb-linux-x86_64-rhel70-3.2.5.tgz

$ mv mongodb-linux-x86_64-rhel70-3.2.5 mongodb-3.2.5

$ mkdir -p /data/mongodbpath/db0 /data/mongodbpath/db1 /data/mongodbpath/db2

$ cd /data/app/mongodb-3.2.5/bin

start three mongodb instance

$ nohup ./mongod –port 10000 –dbpath /data/mongodbpath/db0 –replSet rs0 &

$ nohup ./mongod –port 10001 –dbpath /data/mongodbpath/db1 –replSet rs0 &

$ nohup ./mongod –port 10002 –dbpath /data/mongodbpath/db2 –replSet rs0 &

config mongodb replicat set

$ ./mongo –port 10000

mongo shell > config = {_id: ‘rs0’, members: [ {_id: 0, host: ‘localhost:10000’}, {_id: 1, host: ‘localhost:10001’}, {_id: 2, host: ‘localhost:10002’}]};

mongo shell > rs.initiate(config);

###check rs status and conf
mongo shell > rs.conf()

2. Install and Config ElasticSearch

prepare install

$ cd /data/app

$ wget https://download.elastic.co/elasticsearch/release/org/elasticsearch/distribution/tar/elasticsearch/2.3.1/elasticsearch-2.3.1.tar.gz

$ tar zvxf elasticsearch-2.3.1.tar.gz

$ cd elasticsearch-2.3.1

start elasticsearch server

$ nohup bin/elasticsearch &

$ curl -XGET http://localhost:9200/

3. Install and Config MongoDB river

$ sudo bin/plugin –install com.github.richardwilly98.elasticsearch/elasticsearch-river-mongodb/2.0.9
$ sudo bin/plugin install elasticsearch/elasticsearch-mapper-attachments/3.1.2

4. python with es

install pip

$ wget https://bootstrap.pypa.io/get-pip.py && sudo python get-pip.py && sudo pip install elasticsearch

$ python test_es.py

# url link : https://github.com/elastic/elasticsearch-py
# doc link : http://elasticsearch-py.readthedocs.org/en/master/

from datetime import datetime
from elasticsearch import Elasticsearch
es = Elasticsearch()

doc = {
    'author': 'kimchy',
    'text': 'Elasticsearch: cool. bonsai cool.',
    'timestamp': datetime.now(),
}
res = es.index(index="test-index", doc_type='tweet', id=1, body=doc)
print(res['created'])

res = es.get(index="test-index", doc_type='tweet', id=1)
print(res['_source'])

es.indices.refresh(index="test-index")

res = es.search(index="test-index", body={"query": {"match_all": {}}})
print("Got %d Hits:" % res['hits']['total'])
for hit in res['hits']['hits']:
    print("%(timestamp)s %(author)s: %(text)s" % hit["_source"])

$ python test_es.py

Install spark 1.6.0 with docker

Install docker on cent os 7 with the script

  1. sudo yum update

  2. sudo curl -fsSL https://get.docker.com/ | sh

  3. sudo service docker start

  4. sudo usermod -aG docker yunwei

  5. sudo docker run hello-world

  6. sudo chkconfig docker on

  7. close SELinux

Uninstall

  1. sudo yum list installed | grep docker

  2. sudo yum -y remove docker-engine.x86_64

  3. sudo rm -rf /var/lib/docker

Export docker images from existing images

  1. sudo docker save -o sequenceiq.hadoop-docker.tar 679730efaea5

  2. sudo docker save -o sequenceiq.spark.tar 4f0d42f5530a

Import docker iamges from exported tar file

  1. sudo docker

  2. –input sequenceiq.hadoop-docker.tar

  3. sudo docker tag 789fa0a3b911 sequenceiq/hadoop-docker:2.7.0

  4. sudo docker load –input sequenceiq.spark.tar

  5. sudo docker tag 40a687b3cdcc sequenceiq/spark:1.6.0

  6. sudo docker kill $(sudo docker ps -q) ; sudo docker rm $(sudo docker ps -a -q)

Run hadoop-docker and test

  1. git clone https://github.com/sequenceiq/hadoop-docker.git

  2. sudo cp bootstrap.sh /etc/.

  3. sudo docker run -it sequenceiq/hadoop-docker:2.7.0 /etc/bootstrap.sh -bash

  4. test ,run examples

Run spark-docker and test

  1. git clone https://github.com/sequenceiq/hadoop-docker.git

  2. sudo docker run -it -p 8088:8088 -p 8042:8042 -p 4040:4040 -h sandbox sequenceiq/spark:1.6.0 bash
    or docker run -d -h sandbox sequenceiq/spark:1.6.0 -d

  3. test ,run examples

    $ ./pyspark

Dockerfile for Apache Kafka

The image is available directly from https://registry.hub.docker.com/

Pre-Requisites

  • install docker-compose https://docs.docker.com/compose/install/
  • modify the

Usage

Start a cluster:

Destroy a cluster:

Broker IDs

If you don’t specify a broker id in your docker-compose file, it will automatically be generated based on the name that docker-compose gives the container. This allows scaling up and down. In this case it is recommended to use the --no-recreate option of docker-compose to ensure that containers are not re-created and thus keep their names and ids.

Automatically create topics

If you want to have kafka-docker automatically create topics in Kafka during
creation, a KAFKA_CREATE_TOPICS environment variable can be
added in docker-compose.yml.

Here is an example snippet from docker-compose.yml:

environment:
  KAFKA_CREATE_TOPICS: "Topic1:1:3,Topic2:1:1"

Topic 1 will have 1 partition and 3 replicas, Topic 2 will have 1 partition and 1 replica.
